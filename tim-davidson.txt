    1. What is Semantic HTML? 
      A semantic element clearly describes its meaning to both the browser and the developer
    2. What is HTML used for? 
       formats the text, images, and other media that make up a web page
    3. What is an attribute and where do we put it? 
       Attributes contain extra information about the element that you don't want to appear in the actual content.
    4. What is the h1 tag used for? How many times should I use it on a page?
      To indicate a heading of the highest class. It should only be used once per page. 
    5. Name two tags that have required attributes
        <img> and <a>
    6. What do we put in the head of our HTML document? 
        Links, titles, anything that needs to be communicated but not seen on the rendered page. 
    7. What is an id? 
        defines an identifier (ID) which must be unique in the whole document.
    8. What elements can I add an id to? 
        Any element that you wish to identify for styling. 
    9. How many times can I use the same id on a page? 
        It can only be used once per page. 
    10. What is a class? 
        An attribute that is used to mark for styling. 
    11. What elements can I add a class to? 
        Any element. 
    12. How many times can I use the same class on a page? 
        As often as you like.
    13. How do I get my link to open in a new tab?
        by adding _blank to the link. 
    14. What is the alt attribute in the image tag used for? 
        To explain what the image is in case it does not display or a user is using text reader. 
    15. How do I reference an id?
        #id
    16. What is the difference between a section and a div
        A section is a block level element. A div is just a place holder for later styling. 
    17. What is CSS used for?

        1. Fork your own copy of the repo. 
        2.Clone the repo using the address from the clone button. 
        3. Use terminal to navigate to the place you want it and then use git clone web address to clone. 
        4. cd to the file 
        5. git checkout -b 'firstname-lastname'
        6. git add . 
        7. git commit -m "helpful comment"
        8. git push -u origin branch-Name
        9. in github go to your branch
        10. click pull request
